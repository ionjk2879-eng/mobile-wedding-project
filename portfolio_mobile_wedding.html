Overview

프로젝트 개요

기존 모바일 청첩장 서비스는 사용 기간이 지나면 링크가 만료되고, 하객 RSVP 수집·통계 기능이 별도 도구를 필요로 한다는 불편함에서 시작했다. 에디터에서 만든 청첩장을 영구 보관하고, RSVP 데이터를 실시간 차트로 분석하는 통합 솔루션을 목표로 삼았다.

01
실시간 WYSIWYG 편집
에디터 입력이 폰 프레임 안의 청첩장에 즉시 반영. 에디터·프리뷰 독립 스크롤 지원
02
RSVP 수집 시스템
성함, 측 구분, 동반 인원, 식사 여부, 메시지를 청첩장 내부 폼으로 직접 수집
03
데이터 분석 대시보드
Recharts 기반 도넛 차트·바 차트로 참석 인원, 측별 비중, 식사 선호도 시각화
04
테마 & 폰트 커스텀
warm / dark / midnight 3가지 테마 + 한국어·영문 폰트 6종 실시간 교체

Engineering

기술적 도전 & 해결 과정

에디터 ↔ 프리뷰 독립 스크롤 구현
상황
좌우 분할 레이아웃에서 한쪽을 스크롤하면 반대쪽 또는 body가 함께 스크롤되는 문제
시도
body에 overflow: hidden 적용, 각 패널에 독립 스크롤 부여
해결
에디터·프리뷰 패널 각각에 height: 100vh + overflow-y: auto 명시, body 스크롤 완전 차단
결과
어느 쪽을 스크롤해도 반대쪽이 고정 유지 — 에디터 집중도 향상
다양한 해상도 대응 폰 프레임 스케일링
상황
375×812 고정 폰 프레임이 소형 화면에서 영역을 벗어나거나 너무 작게 보이는 문제
시도
CSS transform: scale() 동적 조정 시도 → 정적 값으로는 대응 불가
해결
기본 scale(0.9) + 미디어 쿼리(≤1000px)에서 scale(1) + 단일 컬럼 레이아웃으로 자동 전환
결과
모든 해상도에서 폰 프레임이 자연스럽게 표시, 모바일 접근성 확보
8개 컴포넌트 단일 상태 동기화
상황
Hero·Greeting·PersonalMessage 등 8개 프리뷰 컴포넌트가 모두 같은 데이터를 바라봐야 하는 구조
해결
App.tsx에 InvitationData 단일 상태 객체 정의, 에디터가 onChange(newData) 콜백으로 전체 교체하는 Unidirectional Data Flow 채택
결과
어느 필드 수정에도 8개 컴포넌트 즉시 동기화, 상태 추적 단순화로 디버깅 비용 절감
