# Sonett - 모바일 청첩장 빌더

## 프로젝트 개요

**Sonett**은 코딩 없이 누구나 아름다운 모바일 청첩장을 제작하고 공유할 수 있는 웹 기반 청첩장 빌더입니다. 실시간 미리보기 에디터에서 테마, 레이아웃, 콘텐츠를 자유롭게 편집하고, 고유 URL로 하객에게 공유할 수 있습니다.

- **프로젝트 유형:** 개인 프로젝트 (풀스택 웹 애플리케이션)
- **개발 기간:** 진행 중
- **배포 환경:** Web (SPA)
- **라이브 데모:** Firebase Hosting 기반 배포

## 주요 기능

### 1. 실시간 WYSIWYG 에디터
- 좌측 에디터 패널에서 콘텐츠를 편집하면 우측 프리뷰에 **즉시 반영**
- 에디터-프리뷰 간 **섹션 동기 스크롤** 지원
- 사이드바 네비게이션으로 15개 편집 섹션 간 빠른 이동
- 전체화면 프리뷰 모드 지원

### 2. 테마 및 디자인 시스템
- **8가지 컬러 테마:** Blush Pink, Champagne, Sage Green, Classic Navy, Burgundy Wine, Lavender, Dusty Rose, Modern White
- **6가지 배경 재질:** 한지, 린넨, 도트, 실크, 수채화 (CSS SVG 필터 기반 구현)
- **7가지 흩날리는 효과:** 벚꽃, 함박눈, 별빛, 나뭇잎, 하트, 반딧불, 꽃가루 (CSS 파티클 애니메이션)
- **4가지 스크롤 등장 효과:** Fade Up, Fade In, Slide In (IntersectionObserver 기반)
- CSS 변수 기반의 테마 시스템으로 일관된 색상 관리

### 3. 메인화면 (Hero) 6가지 레이아웃
- Classic, Overlay, Minimal, Editorial, Fullscreen, Split
- D-Day 자동 카운트다운 표시
- 커스텀 메인 사진 업로드

### 4. 콘텐츠 섹션 (11개)
| 섹션 | 설명 |
|------|------|
| **인사말** | 자유 편집 가능한 인사 메시지 |
| **캘린더** | 예식일 달력 자동 생성, 시간 선택 UI |
| **신랑/신부 한마디** | 프로필 사진 + 개인 메시지 |
| **인터뷰** | Q&A 형식의 신랑/신부 인터뷰 (동적 추가/삭제) |
| **갤러리** | 슬라이드 / 메이슨리 2가지 연출, 라이트박스 뷰어, 터치 스와이프 |
| **타임라인** | 연애~결혼 과정 기록, 사진 첨부, 연도 구분 |
| **장소 및 교통** | Daum 주소검색 → Kakao Maps 지도 연동, 교통정보 자동 생성 |
| **참석의사 (RSVP)** | 참석/불참, 인원수, 식사여부, 메시지 수집 (Firebase 저장) |
| **축의금 계좌** | 3가지 UI 스타일, 계좌번호 복사 기능 |
| **연락처** | 신랑/신부 + 양가 부모님 연락처, 전화/문자 바로 연결 |
| **공유** | 카카오톡 공유, URL 복사 |

### 5. 섹션 순서 관리
- 드래그 없이 버튼으로 섹션 순서를 자유롭게 변경
- 변경된 순서가 프리뷰와 실제 청첩장에 즉시 반영

### 6. 다국어 지원 (한국어/영문)
- 언어 전환 시 이름, 장소 등 주요 정보 영문 별도 입력 가능

### 7. 배경음악
- 내장 프리셋 음악 제공
- 사용자 인터랙션 감지 후 자동 재생
- 플로팅 재생/정지 버튼

### 8. Firebase 연동
- **청첩장 저장/불러오기:** Firestore에 slug 기반 문서 저장
- **RSVP 응답 수집:** 하객 응답 실시간 저장
- **고유 URL 생성:** `/w/{slug}` 형태의 공유 주소

### 9. 관리자 페이지 (`/admin/{slug}`)
- RSVP 응답 목록 테이블 뷰
- 총 참석 인원, 신랑측/신부측 구분, 식사 인원 통계
- 새로고침으로 실시간 데이터 확인

## 아키텍처

```
src/
├── main.tsx                    # 라우팅 설정 (/, /w/:slug, /admin/:slug)
├── App.tsx                     # 에디터 + 프리뷰 메인 레이아웃
├── types.ts                    # TypeScript 인터페이스 정의
├── firebase.ts                 # Firebase 초기화 및 CRUD 함수
├── pages/
│   ├── ViewPage.tsx            # 하객용 청첩장 열람 페이지
│   └── AdminPage.tsx           # RSVP 응답 관리 대시보드
└── components/
    ├── Editor/
    │   └── EditorContainer.tsx # 전체 에디터 UI (1,400+ lines)
    └── Preview/
        ├── Hero.tsx            # 메인화면 (6가지 레이아웃)
        ├── Greeting.tsx        # 인사말
        ├── Calendar.tsx        # 달력
        ├── PersonalMessage.tsx # 신랑/신부 한마디
        ├── Interview.tsx       # 인터뷰 Q&A
        ├── Gallery.tsx         # 갤러리 (슬라이드/메이슨리)
        ├── Timeline.tsx        # 타임라인
        ├── Location.tsx        # 장소 + 카카오맵
        ├── RSVPForm.tsx        # 참석의사 폼
        ├── Money.tsx           # 축의금 계좌
        ├── Contacts.tsx        # 연락처
        ├── Share.tsx           # 공유 기능
        └── ScrollReveal.tsx    # 스크롤 등장 효과 HOC
```

### 페이지 구조

| 경로 | 역할 | 접근 대상 |
|------|------|-----------|
| `/` | 청첩장 에디터 (빌더) | 제작자 |
| `/w/:slug` | 청첩장 열람 | 하객 |
| `/admin/:slug` | RSVP 응답 관리 | 제작자 |

### 데이터 흐름

```
[에디터 입력] → useState(InvitationData) → [실시간 프리뷰 렌더링]
                        ↓ 저장 버튼
              [Firebase Firestore 저장]
                        ↓
              [ViewPage에서 slug로 로드 → 하객에게 동일 UI 렌더링]
                        ↓ RSVP 제출
              [Firestore 서브컬렉션에 응답 저장]
                        ↓
              [AdminPage에서 응답 조회 및 통계]
```

## 기술적 특징

### CSS-in-JS 없이 구현한 테마 시스템
- CSS 변수(`--wedding-main`, `--wedding-bg` 등)로 8개 테마를 클래스 전환만으로 적용
- SVG 필터를 활용한 배경 재질 효과 (한지, 수채화 등)
- CSS `@keyframes` 기반 파티클 애니메이션 (외부 라이브러리 없이 구현)

### 스크롤 기반 인터랙션
- `IntersectionObserver`를 활용한 에디터 섹션 자동 하이라이트
- `ScrollRootContext`를 통한 스크롤 컨테이너 참조 공유
- 스크롤 등장 효과 컴포넌트(`ScrollReveal`) 분리 설계

### 외부 API 연동
- **Daum Postcode API:** 주소 검색 → 예식장 정보 자동 입력
- **Kakao Maps API:** 좌표 변환(Geocoder) + 주변 교통정보 자동 탐색 (지하철, 버스 정류장 검색 및 거리 계산)

### 반응형 에디터 레이아웃
- 데스크탑: 좌측 에디터(700px) + 우측 프리뷰(450px) 분할
- 모바일 프리뷰: 최대 480px 폭 제한으로 실제 모바일 환경 시뮬레이션
